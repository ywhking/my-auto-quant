# Trader Initiator System Description

## Overview

The Trading Initiator receives trading parameters, sends them to QMT Proxy system via WebSocket, and returns execution results.

## Architecture

```
┌─────────────────┐         ┌──────────────┐         ┌─────────────────┐
│ Trading         │         │              │         │ Trading         │
│ Input Source    ├────────►│   QMT Proxy  ├────────►│ Executor       │
│ (Parameters)    │  Order   │              │  Forward   │ (QMT Client)    │
└─────────────────┘         └──────────────┘         └─────────────────┘
       │                                                         │
       └─────────────────────────────────────────────────────────────┘
                         Execution Result (WebSocket)
```

## Main Features

1. **Parameter Input**: Accepts trading parameters via send_order async method
2. **Order Generation**: Formats trading instructions according to specification
3. **WebSocket Communication**: One-time connection per message to QMT Proxy
4. **Result Handling**: Receives and returns execution results with order ID matching
5. **Error Handling**: Logs errors, no retry on connection failures

## API Interface

### send_order Method

**Description:** Async entry point for trading operations. Accepts trading parameters directly.

**Parameters:**
- `stock` (string): Stock code with exchange suffix (e.g., "000001.SH")
- `action` (string): Trading action - `"buy"` or `"sell"`
- `price` (number): Limit order price in currency units
- `number` (number): Order quantity (shares/lots)

**Returns:**
- Success: Execution result dictionary with status and data
- Error: Exception with error message

**Order ID:** Each request carries a unique order ID (UUID format) for matching to response.

**Example:**
```python
result = await send_order(
    stock="000001.SH",
    action="buy",
    price=20.00,
    number=1000
)
# Returns: {"status": "success", "data": {...}, "order_id": "uuid-string"}
```

**Error Handling:**
- Raises `ConnectionError` if cannot connect to proxy (logs error, no retry)
- Raises `TimeoutError` if order execution times out
- Raises `ValidationError` if parameters are invalid

---

## API Interfaces

### 1. Connection to QMT Proxy

**Authentication:** Must obtain token from QMT Proxy first

**Token Request:**
```
GET http://qmt-proxy:8000/token?userName=initiator&password=xxxxxxx
```

**Response:**
```json
{
    "token": "abc123-def456-ghi789"
}
```

### 2. WebSocket Connection

**Endpoint:** `ws://qmt-proxy:8000/ws?token=<token>`

**Connection Role:** `initiator`

**Purpose:**
- Send trading messages to be forwarded to executor
- Receive execution results from executor
- Close connection proactively after receiving response

## Message Formats

### Trading Message Format (Initiator → Proxy → Executor)

```json
{
    "order_id": "550e8400-e29b-41d4-a716-4466554400000",
    "stock": "000001.SH",
    "action": "buy",
    "price": 20,
    "number": 1000
}
```

**Field Descriptions:**
- `order_id` (string): Unique order identifier in UUID format for response matching
- `stock` (string): Stock code with exchange suffix (e.g., "000001.SH" for Shanghai, "000001.SZ" for Shenzhen)
- `action` (string): Trading action - `"buy"` or `"sell"`
- `price` (number): Limit order price in currency units
- `number` (number): Order quantity (shares/lots)

### Execution Result (Executor → Proxy → Initiator)

**Success Response:**
```json
{
    "status": "success",
    "order_id": "550e8400-e29b-41d4-a716-4466554400000",
    "data": {
        "stock": "000001.SH",
        "action": "buy",
        "price": 19.60,
        "number": 1000
    }
}
```

**Error Response:**
```json
{
    "status": "error",
    "order_id": "550e8400-e29b-41d4-a716-4466554400000",
    "message": "Executor not connected",
    "data": {
        "stock": "000001.SH",
        "action": "buy",
        "price": 20,
        "number": 1000
    }
}
```

## Order Flow

### send_order Method Execution

1. **Parameter Validation**: Validate input parameters (stock code, price, quantity)
2. **Order ID Generation**: Generate unique order ID (UUID format) for this request
3. **Authentication**: Obtain token from QMT Proxy `/token` endpoint
4. **Connection**: Establish WebSocket connection to `/ws?token=<token>` (one-time connection per order)
5. **Submission**: Send trading message with order ID via WebSocket
6. **Waiting**: Block and wait for matching execution result
7. **Connection Closure**: Close WebSocket proactively after receiving response
8. **Result Processing**: Handle success/error response
9. **Logging**: Record order submission and result for audit
10. **Return**: Return execution result or raise exception

### Error Handling

**Connection Failure:**
- Log connection error (no retry)
- Raise ConnectionError with error details
- Do NOT retry connection on failure

**Executor Error Response:**
- Receive error response from proxy: `{"status": "error", "message": "...", "order_id": "uuid-string"}`
- Log error details
- Raise RuntimeError with message from proxy
- Do NOT retry to order

**Invalid Order:**
- Validate order before sending
- Raise ValidationError with clear message for invalid parameters

**Timeout:**
- Set timeout for order execution (e.g., 30 seconds)
- Raise TimeoutError if timeout exceeded
- Log timeout event

## Implementation Notes

1. **Order Validation**:
   - Stock code format: `[0-9]{6}\.[A-Z]{2}` (e.g., 600000.SH, 000001.SZ)
   - Price range: Validate against market price limits
   - Quantity: Minimum lot size (usually 100 shares per lot in Chinese market)

2. **Connection Management**:
   - One WebSocket connection per message
   - Close connection proactively after response
   - Generate unique order ID (UUID format) for each request
   - Match response by order ID

3. **Async Pattern**:
   - Use async/await for WebSocket I/O operations
   - `send_order` is an async coroutine
   - No concurrent order submissions (sequential one-by-one)

4. **Logging**:
   - Log all order submissions (timestamp, order details, order_id)
   - Log all execution results (success/failure)
   - Log connection events (connect, disconnect)
   - Log all errors with details
   - No retry logging (since no retry on connection failure)

5. **Configuration**:
   - Proxy URL (host and port)
   - Authentication credentials (username, password)
   - Timeout settings

## Dependencies

- `websockets` - WebSocket client for communicating with proxy
- `pydantic` - Data validation for order structures
- `asyncio` - Async I/O for WebSocket operations
- `logging` - Logging for audit trail
- `uuid` - Generate unique order IDs (UUID format)

## Usage Example

```python
import asyncio
import websockets
import uuid
import logging

logger = logging.getLogger(__name__)

async def send_order(stock: str, action: str, price: float, number: int) -> dict:
    """Execute a trading order asynchronously.
    
    Args:
        stock: Stock code with exchange suffix (e.g., "000001.SH")
        action: Trading action - "buy" or "sell"
        price: Limit order price
        number: Order quantity
    
    Returns:
        Execution result dictionary with status and data
    
    Raises:
        ConnectionError: If cannot connect to proxy (no retry)
        TimeoutError: If order execution times out
        ValidationError: If parameters are invalid
    """
    # 1. Validate parameters
    if not validate_stock_code(stock):
        error_msg = f"Invalid stock code: {stock}"
        logger.error(error_msg)
        raise ValidationError(error_msg)
    if action not in ["buy", "sell"]:
        error_msg = f"Invalid action: {action}"
        logger.error(error_msg)
        raise ValidationError(error_msg)
    if price <= 0:
        error_msg = f"Price must be positive: {price}"
        logger.error(error_msg)
        raise ValidationError(error_msg)
    if number <= 0:
        error_msg = f"Number must be positive: {number}"
        logger.error(error_msg)
        raise ValidationError(error_msg)
    
    # 2. Generate order ID (UUID format)
    order_id = str(uuid.uuid4())
    
    # 3. Get token
    token = await get_token("initiator", "password")
    
    # 4. Connect to WebSocket (one-time connection per order)
    uri = f"ws://localhost:8000/ws?token={token}"
    try:
        async with websockets.connect(uri) as websocket:
            # 5. Send trading message with order ID
            order = {
                "order_id": order_id,
                "stock": stock,
                "action": action,
                "price": price,
                "number": number
            }
            await websocket.send(json.dumps(order))
            logger.info(f"Order sent: order_id={order_id}, stock={stock}, action={action}")
            
            # 6. Wait for result with timeout
            response = await asyncio.wait_for(
                websocket.recv(),
                timeout=30.0
            )
            result = json.loads(response)
            
            # 7. Handle result
            if result["status"] == "success":
                logger.info(f"Order executed: order_id={order_id}, result={result['data']}")
                return result
            else:
                error_msg = f"Order failed: order_id={order_id}, error={result.get('message', 'Unknown error')}"
                logger.error(error_msg)
                raise RuntimeError(error_msg)
                
    except asyncio.TimeoutError:
        error_msg = f"Order timeout: order_id={order_id}, stock={stock}, action={action}"
        logger.error(error_msg)
        raise TimeoutError(f"Order execution timeout after 30 seconds")
    except websockets.exceptions.ConnectionClosed:
        error_msg = f"Connection closed: order_id={order_id}, stock={stock}"
        logger.error(error_msg)
        raise ConnectionError(f"WebSocket connection closed")
    except Exception as e:
        error_msg = f"Unexpected error: order_id={order_id}, {e}"
        logger.error(error_msg)
        raise ConnectionError(f"Connection failed: {e}")

# Helper function: validate stock code
def validate_stock_code(stock: str) -> bool:
    """Validate stock code format.
    
    Args:
        stock: Stock code string
    
    Returns:
        True if valid, False otherwise
    """
    import re
    pattern = r'^[0-9]{6}\.[A-Z]{2}$'
    return bool(re.match(pattern, stock))

# Helper function: get authentication token
async def get_token(username: str, password: str) -> str:
    """Get authentication token from proxy.
    
    Args:
        username: Username
        password: Password
    
    Returns:
        Authentication token string
    """
    import aiohttp
    url = f"http://localhost:8000/token?userName={username}&password={password}"
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            if response.status == 200:
                data = await response.json()
                return data["token"]
            else:
                raise ConnectionError(f"Authentication failed: {response.status}")

# Usage: Submit buy order
try:
    result = await send_order(
        stock="000001.SH",
        action="buy",
        price=20.00,
        number=1000
    )
    print(f"Order executed: {result['data']}")
except ValidationError as e:
    print(f"Validation error: {e}")
except TimeoutError as e:
    print(f"Timeout error: {e}")
except ConnectionError as e:
    print(f"Connection error (no retry): {e}")
except RuntimeError as e:
    print(f"Order execution error: {e}")
except Exception as e:
    print(f"Error: {e}")
```
