整个系统分三部分：

1、交易发起者
交易发起者依据输入信息，生成交易指令发送给QMT代理系统，并等待返回结果
1）和QMT代理系统之间通过websocket建立连接，发送请求，并等待结果
2）交易指令时
  {
    stock: 000001.SH,
    action: buy,
    price: 20,
    number: 1000
  }


2、QMT代理系统
接收交易发起者的交易指令，将指令转发给交易处理者（QMT客户端），接收交易处理者的交易结果，将交易结果返回给交易发起者
1）代理系统和交易处理者直接通过websocket保持长连接，一旦连接中断，将无法处理交易指令


3、交易执行者
接收QMT代理系统发送的交易指令，调用QMT客户端进行下单，并将下单结果返回给QMT代理系统
1）QMT客户端程序需要和QMT代理系统通过websocket保持长连接，并一直监控连接状态，连接故障时需要重连
2）交易结果 
   {
      status: success | error,
      data: {
        stock: 000001.SH,
        action: buy,
        price: 19.60,
        number: 1000
      }
   }
