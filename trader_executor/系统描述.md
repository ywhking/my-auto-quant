# Trader Executor System Description

## Overview

The Trading Executor receives trading instructions from QMT Proxy via WebSocket, calls QMT client to execute orders, and returns execution results back to proxy.

## Architecture

```
┌─────────────────┐         ┌──────────────┐         ┌─────────────────┐
│ Trading         │         │              │         │                 │
│ Initiator       │◄────────┤   QMT Proxy  ├────────►│   QMT Client    │                 │
│ (Parameters)    │  Order   │              │  Order    │   (迅投QMT)     │
│                │              │ Result  │                 │                 │
└─────────────────┘         └──────────────┘         └─────────────────┘
                 │
```

## Main Features

1. **WebSocket Server**: Maintains persistent connection with QMT Proxy
2. **Order Execution**: Calls QMT client API to place buy/sell orders
3. **Status Monitoring**: Continuously monitors connection health
4. **Auto-Reconnection**: Automatically reconnects on connection failure
5. **Result Reporting**: Returns execution status and details to proxy
6. **Callback Integration**: Optional callback for receiving real execution results

## API Interfaces

### 1. Connection to QMT Proxy

**Authentication:** Must obtain token from QMT Proxy first

**Token Request:**
```
GET http://qmt-proxy:8000/token?userName=executor&password=xxxxxxx
```

**Response:**
```json
{
    "token": "abc123-def456-ghi789"
}
```

### 2. WebSocket Connection

**Endpoint:** `ws://qmt-proxy:8000/ws?token=<token>`

**Connection Role:** `executor`

**Purpose:**
- Receive trading messages from proxy to be forwarded to QMT
- Send execution results back to proxy
- Receive real execution results via callback (if configured)

## QMT Client Integration

### QMT Connection

**Initialization:**
```python
from qmt import QMTClient

# Initialize QMT client
qmt_client = QMTClient(
    account_id="your_account_id",
    password="your_password"
)

# Connect to QMT server
qmt_client.connect()
```

**Connection Monitoring:**
- Continuously check connection status
- Implement heartbeat to detect disconnection
- Auto-reconnect on connection loss

### Order Placement

**Buy Order:**
```python
result = qmt_client.buy(
    stock_code="000001.SH",
    price=19.60,
    quantity=1000
)
```

**Sell Order:**
```python
result = qmt_client.sell(
    stock_code="000001.SH",
    price=20.00,
    quantity=1000
)
```

**Order Result Format:**
```python
{
    "order_id": "123456789",
    "status": "success" | "partial" | "failed",
    "filled_quantity": 1000,
    "filled_price": 19.60,
    "message": "Order executed successfully"
}
```

### Query Operations

**Query Position:**
```python
position = qmt_client.query_position(stock_code="000001.SH")
# Returns: {"available": 5000, "frozen": 1000, "total": 6000}
```

**Query Order Status:**
```python
order_status = qmt_client.query_order(order_id="123456789")
# Returns: {"status": "filled", "filled_qty": 1000, "avg_price": 19.60}
```

**Query Account Info:**
```python
account = qmt_client.query_account()
# Returns: {"cash": 100000.00, "market_value": 50000.00, "total": 150000.00}
```

## Message Formats

### Trading Message (Proxy → Executor)

```json
{
    "stock": "000001.SH",
    "action": "buy",
    "price": 20,
    "number": 1000
}
```

**Field Descriptions:**
- `stock` (string): Stock code with exchange suffix
- `action` (string): Trading action - `"buy"` or `"sell"`
- `price` (number): Limit order price
- `number` (number): Order quantity

## Order Flow

### Normal Operation

1. **Authentication**: Obtain token from QMT Proxy `/token` endpoint
2. **Connection**: Establish WebSocket connection to `/ws?token=<token>`
3. **Listening**: Wait for trading messages from proxy
4. **Order Receipt**: Receive trading message from proxy
5. **Validation**: Validate order parameters
6. **Execution**: Call QMT client to place order
7. **Result Processing**: Parse QMT execution result (filled price/quantity)
8. **Response**: Send execution result back to proxy via WebSocket
9. **Monitoring**: Continue monitoring connection health
10. **Connection Closure**: Close WebSocket proactively after response (or keep alive for next message)
11. **Monitoring**: Continue monitoring connection health

### Connection Monitoring & Auto-Reconnection

**Health Check:**
- Send periodic heartbeat messages (ping/pong)
- Check QMT client connection status
- Monitor WebSocket connection state

**Reconnection Logic:**
1. Detect disconnection (WebSocket close or QMT connection loss)
2. Wait with exponential backoff (1s, 2s, 4s, 8s, 16s, max 60s)
3. Reconnect to QMT Proxy WebSocket
4. Reconnect to QMT client API
5. Resume message processing
6. Log reconnection events

**Connection State:**
```python
class ConnectionState(Enum):
    DISCONNECTED = "disconnected"
    CONNECTING = "connecting"
    CONNECTED = "connected"
    RECONNECTING = "reconnecting"
    ERROR = "error"
```

### Error Handling

**QMT Connection Failure:**
- Attempt reconnection with backoff
- Set state to `RECONNECTING`
- Notify proxy (optional): `{"status": "error", "message": "QMT disconnected"}`
- Limit maximum retry attempts
- Log error details

**Order Execution Failure:**
- Parse QMT error message
- Return error response to proxy:
  - `Insufficient funds`
  - `Insufficient position`
  - `Market closed`
  - `Invalid stock code`
  - `Price out of range`
- Log error details

**WebSocket Disconnect:**
- Detect connection close
- Attempt automatic reconnection
- Log disconnect event
- Queue in-progress orders (if any)

**QMT API Error:**
- Catch and handle QMT client exceptions
- Parse error codes and messages
- Return user-friendly error messages
- Retry on transient errors (network timeouts)
- Return user-friendly error messages

## Implementation Notes

### 1. Order Validation

**Before Submitting to QMT:**
- Validate stock code format (`[0-9]{6}\.[A-Z]{2}`)
- Check price range (e.g., minimum tick size)
- Verify quantity is multiple of lot size (usually 100)
- Validate against trading hours (market open/closed)
- Check account balance (for buy orders)
- Check position availability (for sell orders)

### 2. Connection Management

**WebSocket Connection:**
- Use `websockets` library for persistent connection
- Implement ping/pong for connection health
- Set reasonable timeout values
- Handle graceful shutdown

**QMT Client Connection:**
- Initialize QMT client with account credentials
- Maintain connection status flag
- Implement periodic health checks
- Handle QMT-specific connection errors

### 3. Async Pattern

**Concurrency Model:**
- Use async/await for WebSocket I/O
- Use thread pool or separate process for QMT blocking calls
- Avoid blocking WebSocket event loop
- Handle multiple concurrent orders

### 4. Logging

**Log Events:**
- Connection events (connect, disconnect, reconnect)
- Order receipts (order details, timestamp)
- Execution results (success/failure, filled price/quantity)
- QMT errors (error codes, messages)
- System health (CPU, memory, queue size)

### 6. Callback Function

**Description:** Optional callback interface to receive real-time execution results from QMT client.

**Purpose:** The QMT client (迅投XtQuant) provides callback notifications when:
- Order is fully executed
- Partial fill occurs
- Order is rejected

**Callback Events:**
- `on_order_stock(order_id, stock_code, price, volume)`: Order confirmation callback
- `on_order_queue(order_id, stock_code, price, volume)`: Queue notification
- `on_trade_succeeded(order_id, stock_code, price, volume, status)`: Trade completion notification
- `on_order_failed(order_id, error_msg)`: Order failure notification

**Note:** This callback mechanism is optional. If not used, execution status will still be returned to proxy, but real filled price/quantity details will only be available from QMT client logs.

**Implementation:**
```python
from qmt import QMTClient
from trade_callback import MyTradingCallback

# Initialize QMT client with callback
qmt_client = QMTClient(
    account_id="your_account_id",
    password="your_password"
)

# Register callback for real-time notifications
qmt_client.register_callback(MyTradingCallback())

# Connect to QMT
qmt_client.connect()
```
